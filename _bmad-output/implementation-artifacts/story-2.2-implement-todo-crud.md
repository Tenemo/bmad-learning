# Story 2.2: implement todo CRUD

## Goal

Implement validated CRUD endpoints for todos.

## Acceptance criteria

- `GET /api/todos` returns persisted todos
- `POST /api/todos` creates a todo
- `PATCH /api/todos/:id` updates a todo
- `DELETE /api/todos/:id` deletes a todo
- invalid data returns structured `400` responses
